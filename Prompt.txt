# PROMPT – VizPM v2.0 (Visual Project Manager OS)

**Act as a Principal Full‑Stack Software Engineer, Advanced Design System Designer, and Graphical Interface Architect.** You have extensive experience in CAD environments, spatial diagramming, high‑density industrial HMI/SCADA systems, directed‑graph engines, and workflow automation.

Build a highly performant, OS-like web application called **VizPM** (Visual Project Manager). It combines an infinite canvas, Kanban, Gantt, directed graphs, and SCADA-like monitoring. The goal is to explain complex projects at a glance, build them step-by-step, and ensure a **Single Source of Truth**: a change in any view instantly propagates to all others.

---

## 1. ARQUITECTURA GLOBAL (APP SHELL)
La aplicación funciona como un "Sistema Operativo" en el navegador:
- **Header Global Superior**: Logo "VizPM", Menú "Archivo" (Nuevo, Abrir, Guardar, Guardar borrador, Cargar versión), Selectores de Vista Global (Canvas, Gantt, Kanban, Grafo), **Toggle Switch claro** para "Modo Simple/Avanzado", y Avatar de usuario.
- **Taskbar Inferior**: Muestra íconos de las ventanas de proyecto abiertas (minimizar, maximizar, cerrar).
- **Persistencia Global**: `localStorage` guarda configuraciones globales, ventanas abiertas y log de actividad.

## 2. VENTANA DE PROYECTO (CANVAS COMO `main`)
- Cada proyecto es una **Ventana flotante** con barra de título explícita: `[Nombre] [—] [□] [X]`.
- **El Canvas es la tarjeta `main`**: El lienzo infinito es la tarjeta raíz (`id: "main"`, `parentId: null`). Tiene todas las propiedades de una tarjeta y contiene a los macro-bloques como hijos. Tamaño inicial generoso (ej. 8000x5000px), configurable.

## 3. ANATOMÍA Y ESTRUCTURA INTERNA DE TARJETAS (v2.0 REFACTORIZADA)
Las tarjetas tienen jerarquía de 3 niveles (Macro-bloque → Componente → Sub-componente). Su disposición es **altamente configurable** desde el Panel Lateral Avanzado.

### 3.1 Estados de Visualización
- **Modo Colapsado**: Al colapsar (`collapsed: true`), **SOLO se muestra la Zona A unificada**. Todo lo demás se oculta.
- **Modo Compacto (`compactView: true`)**: Ratio 4:3 fijo. Imagen de fondo a pantalla completa con degradado oscuro. Muestra: título (máx. 30 chars), avatares pequeños, mini progreso y botón para volver a detallado.

### 3.2 Zonas del Modo Detallado (Configurables en orden y visibilidad)
Por defecto, se apilan de arriba a abajo, pero el usuario puede reordenarlas u ocultarlas desde el Panel Avanzado:
- **Zona A (Unificada: Título + Progreso)**: *Siempre visible si no está colapsada.*
  - **A1 (Cabecera)**: Fondo oscuro con línea de acento izquierda (`borderColor`). Título (máx. 50 chars, truncado). Derecha: ícono colapsar/expandir y botón "Vista rápida" (ojo). **Prohibido mostrar íconos, WBS o % aquí**.
  - **A2 (Progreso)**: Barra de progreso delgada a lo ancho completo, justo debajo de A1, reflejando `% progress`.
- **Zona B (Media Principal)**: Dedicada a la imagen principal (`image.src`). Configurable para aparecer como banner superior ("top") o columna lateral ("left"/"right").
- **Zona C (Metadatos)**: Fila con avatares de owner, etiqueta de Área, texto de porcentaje, ícono de engranaje (abre panel lateral) y botón "IDEA" (bombilla). 
  - *Nueva característica*: Toggle en panel avanzado para **"Metadatos Compactos"**, que reduce avatares y textos a una sola línea minimalista para ahorrar espacio vertical.
- **Zona D (Pestañas)**: Tabs: **"Trabajo"** (checklist + toggle "Auto-completar"), **"Técnico"**, **"Económico"**, **"Reqs"** y **"Comentarios"**.
- **Zona E (Dropzone de Hijos)**: Contenedor para tarjetas hijas. Texto guía si está vacío.
- **Zona F (Opcional: Galería de Hijos)**: Configurable desde el panel avanzado. Si se activa, muestra una grid (`grid-cols-2` o `3`) con las imágenes principales de las tarjetas hijas, ideal para visualizar sub-componentes de un vistazo.

### 3.3 Configuración de Layout y Zonas (Panel Avanzado)
El Panel Lateral Derecho debe permitir configurar por tarjeta:
- **Orden de Zonas**: Array `zonesOrder` (ej. `["A", "B", "C", "E", "D"]`). El usuario puede hacer drag-and-drop para cambiar el orden de renderizado.
- **Zonas Ocultas**: Array `hiddenZones` (ej. `["D"]` para ocultar pestañas y solo ver la estructura de bloques).
- **Orientación de Hijos (`layoutOrientation`)**: Toggle para forzar a los hijos en la Zona E a mostrarse en modo `"vertical"` (columna) u `"horizontal"` (fila), sobrescribiendo el `layoutMode` por defecto si se desea.

### 3.4 Scroll Adaptativo y Resize
- Si la tarjeta se redimensiona y el contenido excede la altura:
  - La **Zona A** permanece fija (`sticky`, `top: 0`).
  - El resto de zonas visibles obtienen `overflow-y: auto` con scrollbar oculta (`no-scrollbar`).
- **Resize Optimizado**: Usa `transform` / ajustes de tamaño que no fuerzan reflows masivos. `ResizeObserver` *solo* recalcula rutas SVG, **nunca** re-renderiza el DOM interno.

## 4. PANEL LATERAL INTELIGENTE (TRIÁNGULO)
- Botón de **triángulo (▶/◀)** siempre visible en el borde derecho.
- **Auto-expansión** al hacer clic en una tarjeta.
- **Contenido**: Edición de título/descripción, todas las Pestañas, y una sección **"Configuración Avanzada de Tarjeta"** que incluye:
  - Reordenar y ocultar/mostrar Zonas (C, D, E, F).
  - Toggle "Metadatos Compactos" (Zona C).
  - Toggle "Orientación Vertical/Horizontal" de hijos.
  - Toggle "Mostrar Galería de Hijos" (Zona F).
  - Reglas de notificación, permisos, historial de cambios.

## 5. VISTAS FUNCIONALES (SINGLE SOURCE OF TRUTH)
Todas las vistas leen y escriben en el mismo estado. Cambios reactivos e instantáneos:
- **Canvas (SCADA)**: Lienzo infinito, GPU, grilla, conexiones SVG Bézier, minimapa, filtro por owner.
- **Gantt**: Barras horizontales. Arrastrar actualiza fechas globalmente. Dependencias Finish-to-Start y ruta crítica en rojo.
- **Kanban**: Columnas (Sin Empezar, En Progreso, Bloqueado, Completado). Drag cambia `status`.
- **Grafo**: Layout force-directed/jerárquico. Nodos expandibles.
- **Modo Presentación**: Transiciones suaves de cámara.

## 6. DASHBOARD EJECUTIVO (COLAPSABLE)
- Panel en la parte superior de la Ventana de Proyecto.
- **Totalmente colapsable/desplegable** mediante toggle para ganar espacio vertical.
- KPIs en tiempo real: % avance global, Presupuesto vs Gastado (donut), Bloqueos activos, Ruta crítica, Próximos 5 hitos.

## 7. INTELIGENCIA DE DATOS (ROLLUP Y ALERTAS)
- **Rollup Jerárquico**: `presupuesto`/`gastado` del padre = suma de descendientes. `progress` = promedio ponderado de hijos. Valores agregados con ícono "∑". Editar manual rompe agregación (advertencia).
- **Alertas en Cascada**: Monitorea `status: 'Bloqueado'` o `reqs: 'pending'`. Burbujea a la raíz con badge rojo parpadeante (`pulse-border`) en la Zona A/C, incluso colapsado.
- **Notificaciones Inteligentes**: Motor `notificationRules` (ej. `trigger: "dueDate < 48h"`, `action: "badge + webhook"`).

## 8. INTERACCIONES Y PERFORMANCE
- **Drag Optimizado**: Proxy "fantasma" GPU-acelerado. DOM real se actualiza solo en `drop`.
- **Marquee**: Selección múltiple con `requestAnimationFrame` y viewport culling.
- **Drop Zones Inteligentes**: Al arrastrar sobre otra tarjeta, ilumina 3 zonas: Borde superior (hermano anterior), Centro (hijo), Borde inferior (hermano posterior).
- **Atajos**: Ctrl+Z/Y (Undo/Redo, 50 pasos), Ctrl+C/V (Duplicar), Suprimir (con confirmación).

## 9. PERSISTENCIA Y CONTROL DE VERSIONES
- **Claves localStorage**: `vizpm_projects`, `vizpm_open_windows`, `vizpm_global_settings`, `vizpm_activity_log`, `undoStack`/`redoStack`.
- **Auto-save**: Cada 5 min + debounce de 2s tras acciones significativas.
- **Versiones**: "Guardar como borrador" crea snapshot nombrado. "Cargar versión" restaura con confirmación.
- **Nota sobre "IDEA"**: Las notas IDEA **SÍ se guardan en el JSON** del proyecto (campo `ideaNotes: [{text, timestamp}]` dentro de la tarjeta), por lo que se exportan e importan correctamente con el proyecto, a diferencia de las notas temporales de sesión.

## 10. COLABORACIÓN Y ROLES (PREPARACIÓN)
- Modelo de datos con `users: [{name, role}]` y `permissions` por tarjeta.
- Mutaciones estructuradas como operaciones `{ userId, timestamp, targetId, field, value }` (listo para Yjs/CRDT y cola offline).

## 11. MODOS DE VISUALIZACIÓN
- **Simple / Avanzado**: Toggle switch global. "Simple" oculta WBS, rollup, grafo y pestañas técnicas.
- **Focus Modes**: "Trabajo" (oculta hijos, expande tabs), "Bloques" (oculta tabs, expande hijos), "Híbrido" (todo visible).

## 12. ESTILOS Y RESTRICCIONES VISUALES (TAILWIND + DAISYUI)
- **Stack**: HTML5, Vanilla ES6+ JS, **Tailwind CSS y DaisyUI vía CDN** (obligatorio para componentes premium y modo oscuro `data-theme="dark"`).
- **Tema**: Dark mode premium (Vercel/Shadcn). Fondos `#0b0f19`, tarjetas `#111827`, texto blanco, acentos azul/púrpura/verde.
- **UI**: `border-radius` consistente, transiciones 0.2s, imágenes `object-fit: cover`.
- **Animaciones**: Slide-in/slide-out suaves para paneles laterales, dashboard y modales.

## 13. ENTREGABLES
Genera **UN ÚNICO archivo completo y funcional** sin placeholders:
1. **`index.html`**: Contiene todo el HTML, CSS (Tailwind/DaisyUI CDN) y JS (Vanilla ES6+) implementando **cada uno de los 12 puntos anteriores** con precisión milimétrica. 
- **Estado Inicial**: El JS debe incrustar y cargar como estado inicial la estructura del JSON de "AeroVend 1.0" actualizada a v2.0 (incluyendo `zonesOrder`, `ideaNotes`, y la nueva jerarquía de zonas), sirviendo como guía base definitiva.
