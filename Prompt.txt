## 1. SYSTEM RULES & CONSTRAINTS (CRITICAL)
- **NO PLACEHOLDERS**: DO NOT use comments like `// ... implement logic here`, `/* TODO */`, or `// rest of the code`. Write the **complete, working Vanilla JS code** for every feature, especially drag-and-drop, state management, and SVG rendering.
- **SINGLE SOURCE OF TRUTH**: Use a centralized, reactive state manager (e.g., JavaScript `Proxy` or a simple Pub/Sub pattern). Mutating the state must automatically trigger efficient re-renders across Canvas, Gantt, Kanban, and Graph views.
- **MODULAR CODE STRUCTURE**: Organize the JS into clear, distinct blocks: `// --- 1. STATE & STORE ---`, `// --- 2. DATA LOADING & INIT ---`, `// --- 3. RENDER ENGINE ---`, `// --- 4. EVENT LISTENERS & INTERACTIONS ---`, `// --- 5. UTILITIES ---`.
- **CSS EFFICIENCY**: Rely heavily on **Tailwind CSS and DaisyUI** utility classes via CDN to minimize custom CSS. Enforce `data-theme="dark"` on the `<html>` tag. Use **FontAwesome** (or similar CDN) for all interface icons.
- **PERFORMANCE**: Use `requestAnimationFrame` for drag/marquee. Use CSS `transform` for panning/zooming to avoid layout thrashing. `ResizeObserver` must **ONLY** update SVG connection paths, **never** re-render the card's internal DOM.
- **DYNAMIC RENDERING**: When building the card UI, use a dynamic JavaScript loop that iterates through the card's `zonesOrder` array. Only render the zone if it is NOT in the `hiddenZones` array. Do not hardcode the HTML structure of the card with massive if/else chains.
- **NO STANDALONE EXPORT/IMPORT**: Saving always downloads a `.json` file, and opening loads a `.json` file. Do not create separate "Export/Import" buttons.

## 2. GLOBAL ARCHITECTURE (APP SHELL)
- **Top Header**: "VizPM" logo (left), File Menu (New, Open, Save, Save Draft, Load Version), Global View Toggles (Canvas, Gantt, Kanban, Graph), clear Simple/Advanced toggle switch, User Avatar (simulated as "Leo", admin).
- **Bottom Taskbar**: Icons for open project windows. Allows switching, minimizing, or closing them.
- **Global Persistence**: `localStorage` saves global settings, open windows, and activity logs.

## 3. PROJECT WINDOW & CANVAS (`main`)
- Projects open as floating, resizable OS-like windows with a standard title bar: `[Project Name] [—] [□] [X]`.
- **OS Behavior**: Movable (drag title bar), resizable (corners), minimizable to taskbar, maximizable, closable (with save dialog).
- **Canvas as Root**: The infinite canvas is technically the root card (`id: "main"`, `parentId: null`). It holds all project properties (title, desc, tabs, owner, progress, dates) and contains macro-blocks as direct children. Initial size: 8000x5000px (configurable).

## 4. CARD ANATOMY & HIERARCHY (THE CORE)
Strict 3-level hierarchy: Macro-block → Component → Sub-component. Highly configurable via the Advanced Side Panel.
- **Collapsed Mode**: By default, hides Zones B, D, E, and F, showing only Zone A. However, it MUST respect custom user configurations (e.g., keeping Zone C's mini pending blocks visible) as defined in the side panel.
- **Compact Mode (`compactView: true`)**: Fixed 4:3 ratio. Full-cover background image with dark gradient overlay. Shows: truncated title (max 30 chars), mini avatars, mini progress bar, and "Quick view" toggle.
- **Detailed Mode Zones** (Rendered dynamically based on `zonesOrder` array, skipping any zone in `hiddenZones`):
  - **Zone A (Unified Header)**: Dark bg, left accent border (`borderColor`), title (max 50 chars, ellipsis), collapse/expand chevron, "Quick view" eye icon. **STRICTLY PROHIBITED: No icons, WBS codes, or percentages here.**
  - **Zone B (Main Media)**: Primary image (`image.src`). Configurable as top banner, OR left/right column. If left/right, the image takes fixed dimensions and the remaining rows (C, D, E) flow neatly beside it using CSS Flexbox.
  - **Zone C (Metadata)**: Auto-calculated WBS code pill (e.g., "1.2.1"), Area tag (pill), % text, Gear icon (opens side panel), IDEA bulb icon. **Owner avatars**: Google-style circles with initials (max 3 + "+N"). **CRITICAL**: If `owner` is empty or "Sin asignar", display a gray placeholder avatar with "?" and the text "Sin asignar" to prevent app crashes. *Advanced toggle*: "Compact Metadata" shrinks this row to a single minimal line.
  - **Zone D (Tabs)**: "Work" (checklist + "Auto-complete" toggle that sets status to "Completed"), "Technical", "Economic", "Reqs", "Comments" (public thread with avatar, timestamp, text).
  - **Zone E (Child Dropzone)**: Container for child cards. Shows guide text if empty.
  - **Zone F (Optional Child Gallery)**: Advanced toggle to show a grid (`grid-cols-2` or `3`) of child cards' main images.
- **Layout Config**: Per-card `zonesOrder` (e.g., `["A", "C", "E", "D"]`), `hiddenZones` (e.g., `["B"]`), and `layoutOrientation` (force children in Zone E to `"vertical"` or `"horizontal"`).
- **Internal Layout Modes (`layoutMode`)**: `"libre"` (absolute x/y), `"fila"` (horizontal flex), `"columna"` (vertical flex), `"grid"` (auto CSS grid).
- **Adaptive Scroll**: If resized, **Zone A remains sticky** (`top: 0`). Zones C-F get `overflow-y: auto` with a hidden scrollbar (CSS class `.no-scrollbar`).

## 5. SMART SIDE PANEL (TRIANGLE TOGGLE)
- A triangle button (▶/◀) is **always visible** on the right edge of the project window.
- **Auto-expands** when any card is clicked, showing deep-edit properties for that specific card.
- **Contents**: Title/desc inputs, all Tabs, and an **"Advanced Card Config"** section: Zone reordering/hiding, Compact Metadata toggle, Child Orientation toggle, Child Gallery toggle, Notification rules, Permissions, Change history, Detailed connections.

## 6. REACTIVE VIEWS (SINGLE SOURCE OF TRUTH)
- **Canvas (SCADA)**: Infinite pan/zoom, GPU transforms, background grid, animated SVG Bézier connections, minimap, owner filter (dims non-matching cards).
- **Gantt**: Root cards as horizontal bars on a calendar. Drag/resize updates `startDate`/`dueDate` globally. Connections define Finish-to-Start dependencies; critical path is highlighted in red.
- **Kanban**: Columns (Not Started, In Progress, Blocked, Completed). Compact cards. Dragging between columns changes `status` globally.
- **Graph**: Force-directed/hierarchical layout of all cards and connections. Expandable nodes showing title + progress.
- **Presentation Mode**: Smooth camera transitions cycling through root/selected cards (configurable duration, arrow keys to advance, Escape to exit).
- **Camera Bookmarks**: Ability to save named camera positions (X, Y, Scale) in Canvas/Graph views, accessible from a dropdown. Essential for smooth Presentation Mode.

## 7. EXECUTIVE DASHBOARD
- Top panel inside the Project Window, **fully collapsible** via toggle to save vertical space.
- Real-time KPIs: Global % progress (weighted average), Budget vs Spent (donut chart), Active blockers list, Critical path nodes, Next 5 milestones.

## 8. DATA INTELLIGENCE (ROLLUP & ALERTS)
- **Hierarchical Rollup**: Parent `presupuesto`/`gastado` = sum of all descendants. Parent `progress` = weighted average of direct children. Aggregated values show a "∑" icon and italic style. Manual edit of a parent value breaks aggregation (shows warning).
- **Cascading Alerts**: Monitors subtrees for `status: 'Blocked'` or `reqs` with `status: 'pending'`. Incidents bubble up to the root, showing a red blinking badge (CSS `pulse-border`) in Zone C (or overlay in compact mode), even if fully collapsed.
- **Smart Notifications**: `notificationRules` engine in JSON (e.g., `trigger: "dueDate < 48h"`, `action: "badge + webhook"`).

## 9. INTERACTIONS & PERFORMANCE
- **Optimized Drag**: GPU-accelerated "ghost" proxy during drag. Real DOM updates only on `drop`.
- **Marquee Selection**: Semi-transparent blue rectangle. Uses `requestAnimationFrame` and viewport culling to evaluate only visible cards.
- **Smart Drop Zones**: Dragging over a card highlights 3 zones: Top edge (insert as previous sibling), Center (insert as child), Bottom edge (insert as next sibling).
- **Shortcuts**: Ctrl+Z/Y (Undo/Redo, 50 steps), Ctrl+C/V (Duplicate with new ID and offset), Delete (with confirmation dialog: "Promote children to parent level" OR "Delete children entirely").

## 10. PERSISTENCE & VERSIONING
- **localStorage keys**: `vizpm_projects`, `vizpm_open_windows`, `vizpm_global_settings`, `vizpm_activity_log`, `projectVersions`, `undoStack`, `redoStack`, `userNotes`.
- **Auto-save**: Every 5 minutes + 2s debounce after significant actions (drop, resize end, status change).
- **Versions**: "Save as draft" creates a named snapshot in `projectVersions`. "Load version" restores with confirmation. Auto-save does not overwrite named versions.
- **Persistent IDEA Notes**: Saved inside the card's JSON (`ideaNotes: [{text, timestamp}]`), ensuring they export/import with the project (unlike temporary session notes).
- **Activity Feed UI**: A toggle button in the top header (clock/history icon) opens a collapsible right side panel showing the last 50 entries from `vizpm_activity_log`. Clicking an entry automatically switches to the appropriate view, pans the camera to the relevant card, and highlights it.

## 11. COLLABORATION PREP
- Data model includes `users: [{name, role: "admin"|"write"|"read"}]` and optional `permissions` per card.
- State mutations are structured as operations: `{ userId, timestamp, targetId, field, value }` (ready for future Yjs/CRDT integration and offline-first queueing).

## 12. VISUAL MODES & STYLING
- **Simple / Advanced**: Global toggle. "Simple" hides WBS codes, rollup math, Graph view, and technical tabs. "Advanced" shows everything.
- **Focus Modes**: "Work" (hides children, expands tabs), "Blocks" (hides tabs, expands children), "Hybrid" (all visible).
- **Theme**: Premium dark mode (Vercel/Shadcn style). Backgrounds `#0b0f19`, cards `#111827`, crisp white text, accents in blue, purple, and green.
- **UI Details**: `border-radius: 8px`, input backgrounds `#1e293b`, 0.2s transitions. Images use `object-fit: cover` with rounded corners.
- **Animations**: Smooth slide-in/slide-out for side panels, dashboard, activity feed, and modals.

Generate **ONE single, fully functional `index.html` file** without placeholders.
